# Агрегация данных из онлайн-каталогов ритейлеров

Скрипты для массового сбора товаров из каталогов **Магнит** и **Пятёрочка** через консоль браузера.  
Собирают все товары по заданным магазинам и категориям, сохраняют в JSON-файлы.  
Используется для курсовой работы по анализу цен и ассортимента.

## Как пользоваться

1. Откройте сайт в браузере:  
   - Магнит → https://magnit.ru/  
   - Пятёрочка → https://5ka.ru/

2. Откройте консоль разработчика  
   → **F12** → вкладка **Console** (или правой кнопкой → "Просмотреть код" → Console)

3. Вставьте код одного из скриптов (Магнит или Пятёрочка) и нажмите Enter

4. В появившихся prompt-окнах введите:  
   - Магазины через запятую (storeId / shopCode)  
   - Категории через запятую (categoryId)

5. Скрипт автоматически:  
   - покажет прогресс в консоли  
   - скачает JSON-файлы по каждой комбинации магазин+категория  
   - сделает паузы между запросами (чтобы не словить бан)

## Структура файлов

Файлы сохраняются в папку **Загрузки** (Downloads) с именем:
5ka_{storeId}{categoryId}{YYYY-MM-DD}.json
magnit_{storeCode}{categoryId}{YYYY-MM-DD}.json


Примеры:  
- `5ka_324K_251C12891_2026-03-13.json`  
- `magnit_780019_64247_2026-03-13.json`  
- `magnit_729927_41187_2026-03-13.json`

**Содержимое файла** — массив объектов товаров (как приходит от API):

```json
[
  {
    "id": "1000280179",
    "name": "Фарш Мираторг Домашний из свинины и говядины 400г",
    "price": 18999,
    "promotion": { "discountPercent": 20, "oldPrice": 23999, ... },
    "ratings": { "rating": 4.8, "scoresCount": 22106 },
    "gallery": [ ... ],
    "weighted": { ... },
    ...
  },
  ...
]
```


# Как найти storeId и categoryId
## Магнит

  - categoryId — число в начале пути URL после /catalog/
  - Пример: https://magnit.ru/catalog/64247-... → 64247
storeCode — параметр shopCode= в URL
  - Пример: ?shopCode=780019 → 780019
  - Или смотрите в Network → XHR → запрос /goods/search → Payload → "storeCode"

## Пятёрочка

  - categoryId — в Network → XHR → запросы вида
/api/catalog/v2/stores/.../categories/{categoryId}/products
  - Пример: categories/251C12891/products → 251C12891
storeId — в пути URL запроса: /stores/{storeId}/categories/...
  - Примеры: 324K, Y639, A1B2, C7D4
  - Также видно в URL страницы после выбора магазина.

# Требования

  - Современный браузер (Chrome) (На работоспособность в других браузерах не проверял!!)
  - Включённые куки и JavaScript
  - Желательно без VPN (геолокация магазина) (пятерочка блокирует при включенном ВПН!)

## Возможные проблемы и решения

| Проблема                              | Решение                                                                 |
|---------------------------------------|-------------------------------------------------------------------------|
| 🚫 HTTP 403/429 (блокировка)          | Подождать 5–10 минут, собирать меньше категорий/магазинов за раз       |
| ⚠️ 0 товаров в файле                  | Проверить правильность storeId / categoryId                             |
| ❓ Запросы не выполняются              | Убедитесь, что находитесь на сайте ритейлера (не на другой вкладке)    |
| 💾 Файлы не скачиваются               | Отключить блокировщик рекламы / попапов в браузере                     |
